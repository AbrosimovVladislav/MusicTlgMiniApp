# Music Expert App — Product Flow Document (MVP)

---

## 1. Концепция продукта

**Music Expert App** — Telegram Mini App маркетплейс, где начинающие криэторы (музыканты, продюсеры и другие) могут получить платную консультацию у профессионального эксперта в своей области.

**Ключевая механика:** двусторонний матчинг между пользователем (заказчиком) и экспертом — сделка происходит только при взаимном интересе обеих сторон.

**Платформа:** Telegram Mini App (Mobile)

---

## 2. Роли в системе

| Роль | Описание |
|---|---|
| **User (Пользователь)** | Начинающий криэтор, которому нужна консультация |
| **Expert (Эксперт)** | Профессионал в своей области, оказывающий платные консультации |

---

## 3. Авторизация

Отдельная регистрация **не нужна**. При открытии Mini App берём данные из Telegram-профиля через `initData`:
- `telegram_user_id` — уникальный идентификатор
- `first_name`, `last_name`, `username`, `photo_url`

Онбординг сводится только к **выбору роли**.

---

## 4. User Flow — Пользователь

### 4.1 Онбординг
- Сплэш-скрины (1–2 экрана) — краткое описание ценности приложения
- Выбор роли: **Пользователь** или **Эксперт**

---

### 4.2 Главная страница (Home)
После онбординга пользователь попадает на главный экран.

**На экране:**
- Кнопка / CTA **«Создать новый запрос»**
- Список активных запросов пользователя (если есть)
- Статус каждого запроса

---

### 4.3 Flow создания запроса

**Шаг 1 — Описание запроса**
- Текстовое поле: в чём суть запроса

**Шаг 2 — Категория и подкатегория**
- Выбор категории и подкатегории

**Шаг 3 — Дополнительные параметры** *(опционально в MVP)*
- Уровень опыта
- Бюджет *(пользователь может редактировать позже)*

**Шаг 4 — Подтверждение**
- Превью запроса
- Кнопка **«Опубликовать запрос»**

---

### 4.4 Страница деталей запроса

Пользователь может в любой момент зайти в свой запрос из Home и увидеть:
- Текст, категорию, статус
- Возможность редактировать (описание, бюджет)
- Список экспертов по запросу с фильтрацией:
  - Все доступные по категории
  - Лайкнутые пользователем
  - Лайкнувшие пользователя (отклики экспертов)
  - Взаимный матч

---

### 4.5 Список экспертов (Matching Screen)

После публикации запроса — экран подбора экспертов.

- Список экспертов, чьи категории совпадают с запросом
- Карточка: фото, имя, специализация, краткое описание
- По нажатию — детальная страница эксперта (попап или отдельный экран):
  - Фото, имя, специализация, описание, категории, рейтинг *(будущее)*
- ❤️ **Лайк** — пользователь заинтересован в эксперте (можно лайкнуть нескольких)

---

### 4.6 Экран ожидания (Pending)

Пользователь переходит в статус ожидания — видит список лайкнутых экспертов.

Статусы на карточке эксперта:
- **Ожидание** — лайк отправлен, ответа нет
- **Откликнулся** — эксперт проявил интерес (написал первым)
- **Матч** — взаимный интерес, можно переходить к оплате

---

### 4.7 Отдельный экран поиска экспертов

Независимый от запросов экран — просмотр и поиск всех экспертов.
- Фильтрация по категории / подкатегории
- Сортировка *(параметры уточнить позже)*
- Из этого экрана также можно лайкнуть эксперта и привязать к запросу

---

### 4.8 Оплата и получение контактов (Target Action — MVP)

При взаимном матче:

1. Экран **«Матч с экспертом [Имя]»**
2. Кнопка **«Оплатить консультацию»**
3. Варианты оплаты:
   - **Telegram Stars** — нативная оплата (MVP, реализовано)
   - **Через провайдера** — экран готов, редирект замокан (реальный провайдер подключается позже)
4. После оплаты — пользователь получает **Telegram username эксперта**
5. Дальнейшее общение напрямую в Telegram *(MVP)*

## 5. Expert Flow — Эксперт

### 5.1 Онбординг эксперта
- Сплэш-скрины (общие)
- Выбор роли: **Эксперт**
- Заполнение профиля (обязательный шаг перед входом в приложение)

---

### 5.2 Заполнение профиля

**Обязательные поля:**
- Фото профиля *(берётся из Telegram, можно заменить)*
- Имя, фамилия
- Описание опыта
- Категории и подкатегории работы
- Стоимость консультации *(фиксированная или задаётся экспертом)*
- Telegram username *(хранится в системе, передаётся пользователю после оплаты)*

**Опционально (закладывается):**
- Портфолио / ссылки на работы

---

### 5.3 Главная страница эксперта (Home)

**Два раздела:**

**Раздел 1 — «Запросы по моим категориям»**
- Все активные запросы пользователей, категории которых совпадают с категориями эксперта
- Карточка: описание, категория/подкатегория, дата
- Эксперт может открыть запрос и **откликнуться**

**Раздел 2 — «Меня лайкнули»**
- Запросы, где пользователь уже лайкнул этого эксперта
- Приоритетный раздел — здесь уже есть интерес со стороны пользователя

---

### 5.4 Логика отклика

**Сценарий A — Эксперт откликается первым:**
1. Нажимает **«Откликнуться»** на запрос
2. Пользователю приходит уведомление от бота
3. Пользователь видит эксперта в статусе «Откликнулся» — может лайкнуть в ответ
4. Взаимный лайк → матч → оплата

**Сценарий B — Пользователь уже лайкнул эксперта:**
1. Эксперт видит запрос в разделе «Меня лайкнули»
2. Нажимает **«Принять»**
3. Немедленный матч → уведомление пользователю → оплата

---

### 5.5 После матча

- Эксперт получает уведомление об оплате от бота
- После оплаты его Telegram username передаётся пользователю
- Статус запроса → **«В работе»**

---

## 6. Статусы запроса

### Статус жизни (status)
| Статус | Описание |
|---|---|
| `draft` | Запрос создан, но не опубликован |
| `published` | Запрос опубликован, виден экспертам |
| `matched` | Найден взаимный матч, ожидается оплата |
| `in_progress` | Оплата прошла, консультация в процессе |
| `completed` | Завершён |

### Флаг активности (is_active)
| Значение | Описание |
|---|---|
| `true` | Запрос активен, виден экспертам |
| `false` | Запрос неактивен (истёк или закрыт вручную) |

**Срок жизни:** 90 дней с момента публикации → `is_active = false`. Пользователь видит запрос в своём списке, эксперты — нет.

Пользователь может иметь **несколько активных запросов** одновременно.

---

## 7. Система уведомлений

**Механика:** Telegram Bot API.
1. При открытии Mini App — `requestWriteAccess()` запрашивает у пользователя разрешение на получение сообщений от бота
2. На бэкенде — бот шлёт сообщения через `sendMessage` при наступлении событий
3. Никаких внешних push-сервисов не нужно — всё нативно в Telegram

| Триггер | Получатель | Текст уведомления |
|---|---|---|
| Эксперт откликнулся | Пользователь | «Эксперт [Имя] заинтересован вашим запросом» |
| Взаимный матч | Пользователь + Эксперт | «У вас новый матч с [Имя]!» |
| Оплата прошла | Эксперт | «Пользователь оплатил консультацию» |
| Получены контакты | Пользователь | «Контакты эксперта [Имя] доступны» |

## 9. Ключевые экраны

### User-сторона
1. Сплэш / Онбординг
2. Выбор роли
3. Home (список запросов + CTA)
4. Создание запроса (Step 1–4)
5. Список экспертов по запросу (Matching Screen)
6. Детальная карточка эксперта (попап)
7. Страница деталей запроса (статус, эксперты, редактирование)
8. Поиск и просмотр всех экспертов (с фильтрами)
9. Экран матча + оплата
10. Экран «Контакты получены»

### Expert-сторона
1. Онбординг + заполнение профиля
2. Home эксперта (2 раздела)
3. Детальная карточка запроса + отклик
4. Экран матча / подтверждения
5. Уведомление об оплате

## 10. Out of Scope (MVP) — точки роста

- Встроенный чат / звонок внутри приложения
- Рейтинги и отзывы экспертов
- Фильтры и сортировка экспертов (частично в поиске)
- История завершённых консультаций
- Реферальная программа
- Верификация экспертов
- Реальный платёжный провайдер (экран готов в MVP, редирект замокан)
- Уведомления о просроченных запросах (90 дней)
